About the Benchmark Data

All benchmarks run against a synthetic enterprise knowledge graph generated by RCRT's datagen engine: 30 employees, 8 projects, 4 organizations, 6 office locations, with interconnected relationships (reporting chains, project assignments, skills, budgets, divisions). Entity names are fictional (e.g. Alaric Ashford, Project Zephyr, Verdant Systems).

The knowledge is seeded into isolated benchmark workspaces as RCRT breadcrumbs with LOD resolution layers (R0–R3). Both RCRT and the traditional RAG baseline search the same Vertex AI hybrid vector index over the same data — the only difference is how context is assembled and delivered to the model. Questions are generated deterministically from the entity graph with verified ground-truth answers.

Each surface uses the same benchmark workspace and the same seeded data. The difference is purely in the retrieval and context assembly pipeline:

  • RCRT — Full LOD18 multi-factor context cascade with structured memory, multi-hop entity resolution, scope inheritance, and resolution-adaptive rendering
  • Vertex AI Search — Same hybrid search index, but flat top-K context stuffing with no multi-hop, no resolution layers, no scope awareness
  • Direct (control) — Model receives only the question, no retrieved context at all
